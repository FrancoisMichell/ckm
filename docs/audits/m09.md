## Phase 1 ‚Äî Bugs

### P1-M9-1 (low) ‚Äî `teacher-isolation.e2e-spec.ts` mutates shared `World` mid-suite
The `/class-sessions/:id/end` cross-teacher test starts and ends Teacher B's `sessionB`
as a side effect (lines 343-351). The `World` is documented as "built once... each suite
reads from it (no mutations)" but this test transitions `sessionB` through start‚Üíend. The
cleanup is best-effort and runs only if the assertion path is reached; a failure earlier in
the `it` leaves `sessionB` started/ended. Later isolation assertions only check IDs and
`deleted_at`/`status`/`notes` of `sessionB`, so today it does not cause flake ‚Äî but the
"no mutations" contract is already broken and a future added assertion on `sessionB.start_time`
would flake. Recommend a dedicated throwaway session for the start/end isolation case.

### P1-M9-2 (low) ‚Äî Coverage gate ignore-list breadth not verifiable from diff alone
Plan 9.6 specifies ignoring module/entity/DTO/setup-app/main from coverage. The branch thresholds
(75) plus a broad ignore list can mask untested branches in services. Not a defect per se, but
confirm the `collectCoverageFrom`/`coveragePathIgnorePatterns` does not also exclude `*.service.ts`
or `*.filter.ts` ‚Äî those are the files where the 75% branch floor must bite. (Could not see the
jest config contents in this pass; flagging for the user to spot-check.)

### P1-M9-3 (info) ‚Äî `findExistingAttendance` ignores soft-deleted rows by design
`AttendancesService.findExistingAttendance` / `getExistingOrThrow` query `{ sessionId, studentId }`
without `withDeleted`. This is correct (a soft-deleted attendance should not block a fresh insert,
and the partial unique index is `WHERE deleted_at IS NULL`). Recording it as understood, not a bug ‚Äî
the idempotency invariant is upheld at both the app and DB layer.

## Phase 2 ‚Äî Security

### P2-M9-1 (info) ‚Äî Teacher-isolation suite is genuinely comprehensive
9.3 covers students, classes, class-sessions, attendances across GET-list/GET-one/PATCH/DELETE/
restore/by-class/by-teacher/by-date-range/start/end/notes/present/late/absent/excused/POST/bulk,
and additionally asserts non-mutation of the foreign row (status/notes/deleted_at unchanged) on the
404 path. Every cross-teacher path asserts `404` and explicitly `not 403`. This satisfies the
release-blocker invariant at the e2e layer. The code-level guarantee is verified separately in the
tag audit (TAG-2).

### P2-M9-2 (low) ‚Äî Throttle bypass token gate is fail-closed but env-coupled
`ThrottlerModule.skipIf` requires BOTH `NODE_ENV === 'test'` AND a matching
`x-test-skip-throttle` header equal to `THROTTLE_TEST_BYPASS_TOKEN` (‚â•16 chars, optional in Joi,
absent in prod ‚Üí header compared to `undefined` ‚Üí false). This is correctly fail-closed. Residual
risk: a deploy that erroneously sets `NODE_ENV=test` AND leaks the bypass token would be bypassable.
Acceptable given the double gate; recommend documenting in `accepted-findings.md` so it is not
re-raised.
